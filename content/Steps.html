Steps in FFMPEG/SRT implementation

# Get the ffmpeg plugin project from Perforce

# Make a subsystem

To wrap access to ffmpeg
Maybe a singleton
(Maybe in BeginPlay) an actor can get a reference to a ffmpeg interface which runs on worker thread

# Make an input queue

Similar to https://github.com/TimmHess/UnrealImageCapture where textures which have been retrived from the viewport or camera can be pushed on a queue

A worker thread can poll the queue or react to it being updated (using events) and get a job from the queue

Jobs would have:

  • a texture
  • a time captured
  • maybe a sequence number
  • maybe a destination to support multiple outputs

# Worker thread processing

Use ffmpeg to

  • read input texture
  • determine format
  • decode format
  • decode raw data
  • encode to h.264
  • encode to mpegts container

# ffmpeg state

For each destination there would be:

  • the srt url
  • maybe a file url alternative
  • ffmpeg objects for the container and output streams and associated contexts

# Blueprint access object

Have an FFMPEG object with these functions:

  • configure - would take an FFMPEG Configuration node
  • start queue processing
  • clear queue
  • stop queue processing
  • to add a texture

Have an FFMPEG Configuration node which would configure the

  • input container format
  • input data format
  • conversion parameters - presets
  • output data format (h.264 h.265)
  • output container format (mpegts)
  • output destination - srt or file

# nice to haves

  • integration into Unreal stats for performance timings
  • proper integration with Unreal threading
